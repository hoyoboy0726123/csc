# 🎙️ AI 語音驅動全自動開發系統 (V2C - Groq 版)

這是一個將「口述想法」直接轉化為「可執行專案」的自動化流水線。本系統優化了長上下文處理與自動化續傳，是目前最穩定的 AI 代碼生成解決方案之一。

---

## 🏆 核心建議：使用模型
為了獲得最佳的長代碼生成品質與全局架構視野，我們強烈建議在系統設定中選擇：
- **首選模型**：**`kimi-k2-instruct-0905`**
- **推薦理由**：具備 **256K** 超長上下文視窗，能同時理解上百個專案檔案的關聯性，有效減少引用錯誤。

---

## 🛠️ 環境安裝指南

### 方法 A：使用 `uv` 安裝 (推薦 - 極速且穩定)
[uv](https://github.com/astral-sh/uv) 是目前最強大的 Python 套件管理工具。
```cmd
# 1. 初始化並同步環境 (自動讀取 pyproject.toml)
uv sync

# 2. 啟動專案
uv run streamlit run app.py
```

### 方法 B：使用傳統 `pip` 安裝
```cmd
# 1. 建立並啟用虛擬環境
python -m venv .venv
.venv\Scripts\activate

# 2. 安裝所有依賴
pip install -r requirements.txt

# 3. 啟動專案
streamlit run app.py
```

## 📖 詳細啟動與使用說明書

### Step 1：系統設定 (首次使用必看)
1.  啟動應用程式後，進入左側選單的 **「設定系統」**。
2.  **配置 Groq Key**：填入您的 Groq API Key 並點擊「測試並儲存」。
3.  **選擇主模型**：在下拉選單中選取 `kimi-k2-instruct-0905`。
4.  **語音設定**：預設建議選取 `Groq-Whisper` 以獲得最快的轉譯速度。

### Step 2：需求擷取與 PRD 生成
1.  進入 **「需求擷取」** 頁面。
2.  **錄音**：點擊 `⏺️ 開始錄音` 口述您的功能想法，完成後點擊 `⏹️ 停止並轉錄`。
3.  **微調**：文字會自動填入編輯框，您可以手動修正或刪除冗長內容。
4.  **生成 PRD**：點擊 `✨ 產出高品質 PRD`。AI 會自動過濾閒聊雜訊，產出技術核心文件。
5.  **滾動修正**：若對 PRD 不滿意，可在下方輸入指令（如：「多加一個登入功能」）讓 AI 進行增量更新。

### Step 3：一鍵分段生成程式碼
1.  確認 PRD 無誤後，點擊 **「🚀 一鍵分段生成完整專案」**。
2.  **進度追蹤**：系統會顯示 `(1/36)` 等進度標示，並依序完成架構規劃、合約定義與代碼填充。
3.  **TPD 續傳**：若觸發每日配額限制，畫面會出現 **倒數計時器**。**請勿關閉視窗**，倒數結束後系統會自動從斷點續傳。

### Step 4：專案總覽與 AI 助手
1.  進入 **「專案總覽」**。
2.  **檔案編輯**：左側可瀏覽並直接修改代碼（點擊「✏️ 進入編輯模式」）。
3.  **對話開發**：右側為 AI 助手聊天室。您可以直接問它：「為什麼這個 API 報錯？」或指令「幫我把資料庫改為 MongoDB」，AI 會自動分析全案並套用變更。

## ⚠️ 速率限制 (Rate Limits) 處理說明
Groq 免費層有嚴格的限制：
- **RPM (15次/分)**：系統內建滑動窗口，每 14 次請求會自動休息 60 秒。
- **TPD (每日總量)**：若觸發此限制，請觀察畫面上的等待時間。這通常是 24 小時滑動重置，等待幾分鐘到幾十分鐘即可自動恢復。
